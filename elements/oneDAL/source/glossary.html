Glossary¶

Machine learning terms¶

Categorical feature

A feature with a discrete domain without ordering between values.

Synonyms: nominal feature.

Classification

A supervised machine-learning problem of assigning discrete labels (categorical or ordinal) to feature vectors.

Clustering

An unsupervised machine-learning problem of grouping feature vectors onto (usually categorical) groups without prior information on them.

Continuous feature

A feature with values in a domain of real numbers.

Synonyms: quantitative feature, numerical feature.

Dataset

A collection of observations.

Feature

A particular property or quality of a real object or an event. Has a defined type and domain.

Synonyms: attribute, variable

Feature value

A single point in the domain of the feature.

Feature vector

A vector that encodes information about single real object or an event. Contains at least one feature value.

Inference

A process of applying a trained model to the inference set with a goal to make outputs based on input feature vectors.

Synonyms: prediction.

Inference set

An unlabeled dataset used at the inference stage to make an outputs from the model.

Label

A feature of some object or event which value is target to be predicted in supervised learning task. On the training stage, the model which finds the best dependency between input feature vector and labels is selected. Label is one of outputs can be made by model on the inference stage.

Model

A result of training machine-learning algorithm that stores information necessary to run inference on a new dataset.

Observation

A vector of at least one feature value and zero or more label values. Each of labels and features can encode different objects or events.

Synonyms: instance, sample.

Ordinal feature

A feature with a discrete set of possible values and some ordering between them.

Regression

A supervised machine-learning problem of assigning continuous labels to feature vectors.

Response

A value of a single label. For example, response may be a flag indicating that an observation belongs to a particular class.

Supervised learning

Training process with labeled training set. The best model of dependencies between input feature vector and labels is selected.

Training

A process of creating a best model (in terms of some quality criteria) based on information extracted from a training set.

Training set

A dataset used at the training stage to create a model.

Unsupervised learning

Training process where training set does not contain information of dependencies between feature vectors and outputs need to be produced by the model.

oneDAL terms¶

Batch Mode

The computation mode for an algorithm in oneDAL, where all the data needed for computation is available at the start and fits the memory of the device on which the computations are performed.

Contiguous data

A special property of the data format: the data are stored as one contiguous memory block.

Data format

The representation of internal structure of the data: this information is enough to efficiently access them.

Data type

An integral or floating-point type which is used to physically store the data.

Homogeneous data

A special property of the data format: all the features have the same type and domain.

Immutability

The object is immutable if it is not possible to change its state after creation.

Online Mode

The computation mode for an algorithm in oneDAL, where the data needed for computation becomes available in parts over time.

Reference-counted object

A oneDAL object which stores the number of references to the unique implementation. Such object has a lightweight copy constructor and assignment operator, which simply increases the number of references. Implementation is automatically freed when the number of references becomes equal to zero.

Workload

A task of applying a oneDAL algorithm to a dataset.

Common oneAPI terms¶

API

Application Programming Interface

DPC++

Data Parallel C++ (DPC++) is a high-level language designed for data parallel programming productivity. DPC++ is based on SYCL* from the Khronos* Group to support data parallelism and heterogeneous programming.

Host/Device

OpenCL [OpenCLSpec] refers to CPU that controls the connected GPU executing kernels.

JIT

Just in Time Compilation — compilation during execution of a program.

Kernel

Code written in OpenCL [OpenCLSpec] or SYCL and executed on a GPU device.

SPIR-V

Standard Portable Intermediate Representation - V is a language for intermediate representation of compute kernels.

SYCL

SYCL(TM) [SYCLSpec] — high-level programming model for OpenCL(TM) that enables code for heterogeneous processors to be written in a “single-source” style using completely standard C++.
